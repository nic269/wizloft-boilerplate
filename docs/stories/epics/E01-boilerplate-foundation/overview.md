# Overview

## Current Behavior

The repository starts as a Harness seed with no application implementation.

## Target Behavior

The repository becomes a runnable personal SaaS boilerplate monorepo with generic app surfaces, packages, provider
abstractions, docs, templates, and validation commands.

## Affected Users

- Owner/developer starting future apps from the boilerplate.
- Future agents implementing product-specific templates or stories.

## Affected Product Docs

- `SPEC.md`
- `docs/product/boilerplate-platform.md`
- `docs/decisions/0008-personal-saas-boilerplate-architecture.md`

## Non-Goals

- Complete production auth/invite UI flows.
- Domain templates with real LMS/dev-tools/Shopify business models.
- Copying Ops Hub domain code.
